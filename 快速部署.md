# K8s Dashboard 快速部署指南

## 一键部署

### 1. 打包（本地）

```bash
bash build.sh
```

### 2. 上传到服务器

```bash
scp k8s-dashboard-release.tar.gz root@your-server:/tmp/
```

### 3. 安装（服务器）

```bash
cd /tmp
tar -xzf k8s-dashboard-release.tar.gz
cd k8s-dashboard-release
sudo ./install.sh
```

### 4. 访问

```
http://your-server-ip:8000
```

## 目录结构

```
/opt/k8s-dashboard/          # 安装目录
├── main.py                  # FastAPI 应用入口
├── routes.py                # API 路由
├── cluster_routes.py        # 集群管理路由
├── k8s_service.py           # K8s 服务封装
├── cluster_manager.py       # 集群管理器
├── database.py              # 数据库管理
├── schemas.py               # Pydantic 模型
├── mock_data.py             # 模拟数据
├── config.py                # 配置管理
├── venv/                    # Python 虚拟环境
├── static/                  # 前端静态文件
├── kubeconfigs/             # Kubeconfig 存储目录
└── .env                     # 环境配置
```

## 常用命令

```bash
# 查看服务状态
systemctl status k8s-dashboard

# 查看日志
journalctl -u k8s-dashboard -f

# 重启服务
systemctl restart k8s-dashboard

# 停止服务
systemctl stop k8s-dashboard
```

## 首次使用

1. 访问 `http://your-server-ip:8000`
2. 点击右上角「集群管理」
3. 点击「添加集群」
4. 填写集群名称，粘贴 kubeconfig 内容
5. 点击确定，完成配置

---

## 环境要求

- Ubuntu 18.04+ / Debian 10+ / CentOS 7+ / RHEL 8+
- Python 3.8+
- 内存 512MB+
- 磁盘 1GB+

---

## 端口

- 默认：8000
- 修改：编辑 `/opt/k8s-dashboard/.env` 中的 `PORT` 值

---

## 故障排查

### 图表不显示

- 确保已添加并选择集群
- 刷新页面数据
- 检查浏览器控制台是否有错误

### 服务无法访问

- 检查防火墙：`sudo ufw status` 或 `sudo firewall-cmd --list-all`
- 检查服务状态：`systemctl status k8s-dashboard`
- 查看日志：`journalctl -u k8s-dashboard -n 50`

### K8s 连接失败

- 通过集群管理页面重新上传 kubeconfig
- 检查 kubeconfig 格式是否正确
- 确认 K8s API Server 可访问
